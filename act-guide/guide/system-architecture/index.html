系统架构 ​

下面讲重点讲解互动整体架构、互动数据流转、互动生命周期、逻辑组件等。

整体架构图 ​

互动数据流转 ​

使用 MVVM 模式。Model将数据差异抹平，给ViewModel提供数据源，ViewModel处理Http、Pomelo消息，更新数据，体现到View。 最终无论是什么互动，都产生一份统一的数据结构。

提示

本次重构，重点工作就是将数据结构统一化。数据统一是互动的基石，未来的所有扩展都得益于此。因数据的统一才会有后面的互动生命周期和逻辑组件等。

互动生命周期 ​

为了方便管理单个互动，我们将互动分为三个阶段：互动列表页、互动自定义配置页（可选）、互动作答页。

封装用户行为钩子：beforeEnter、onEnter、beforeLeave、onLeave。

互动关键事件钩子： onActStart、onActRestart、onActFinish。

互动生命周期工作流程如下：

点我查看代码
typescript
/**
 * 生命周期钩子参数
 */
export interface HookParams {
  from: EnumProcessType
  to: EnumProcessType
}

/**
 * 互动生命周期
 */
export interface IActLifecycle {

  /**
   * 声明周期钩子🪝 - 离开阶段前
   */
  beforeLeave(params: HookParams): boolean | Promise<boolean>

  /**
   * 生命周期钩子🪝 - 离开阶段
   */
  onLeave(params: HookParams): void

  /**
   * 生命周期钩子🪝 - 进入阶段前
   */
  beforeEnter(params: HookParams): boolean | Promise<boolean>

  /**
   * 生命周期钩子🪝 - 进入阶段
   */
  onEnter(params: HookParams): void

  /**
   * 渲染模板
   */
  render(): CtrlRenderTemplate

  /**
   * 切换到下一个阶段
   */
  next(): Promise<void>

  /**
   * 切换到上一个阶段
   */
  prev(): Promise<void>

  /**
   * 切换到指定阶段
   */
  goto(to: EnumProcessType, ignoreHooks: boolean): Promise<void>

  /**
   * 状态恢复
   */
  recovery(statistic: StandardStatistic): void

  /**
   * 监听消息 —— 互动开始
   */
  onActStart(): void

  /**
   * 监听消息 —— 互动结束
   */
  onActStop(mode: ActServer.Statistic.FinishType): void

  /**
   * 监听消息 —— 互动完成
   */
  onActFinish(mode: ActServer.Statistic.FinishType): void
}
